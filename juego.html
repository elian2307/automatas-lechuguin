GUÍA DE CONSULTA PARA REPASO

1. Análisis Semántico

Verifica que los operandos sean compatibles con los operadores lógicos. Errores comunes:

  • Asignar una cadena a un entero: int x = "hola".
  • Suma de tipos incompatibles sin cast explícito en lenguajes fuertemente tipados.
  • Uso de variables que no fueron declaradas previamente.

2. Código Intermedio y Notación Polaca

La Notación Polaca Inversa (Postfija) coloca el operador después de los operandos. Es ideal para evaluar con pilas:

  • A + B → A B +
  • A + B * C → A B C * + (ya que la multiplicación tiene precedencia)
  • (A + B) * C → A B + C *

Cuádruplos: Compuestos por (Operador, Operando1, Operando2, Resultado) utilizando variables temporales explícitas (e.g. t1).

3. Optimización Peephole (Mirilla)

Busca secuencias cortas de código ensamblador redundantes o simplificables:

  • Redundancia de Cargas: MOV R1, R2 seguido de MOV R2, R1 (la segunda es innecesaria).
  • Álgebra trivial: Cambiar ADD R1, R1, #0 por nada (no hace nada).
  • Reducción de Fuerza: Cambiar MUL R1, R2, #2 por un desplazamiento de bit a la izquierda.

4. Segmentos de Memoria

  • STACK: Almacena variables locales locales al ámbito y registros de activación de funciones en ejecución.
  • HEAP: Almacena memoria asignada dinámicamente en tiempo de ejecución (ej: malloc, new) administrada por el programador.
  • DATA / BSS: Almacena variables globales e inmutables del programa.
  • CÓDIGO (TEXT): Almacena las instrucciones compiladas en lenguaje máquina directamente legibles por el procesador.
